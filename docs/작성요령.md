## 3. 섹션별 작성 지침 (★ 우리 프로젝트 내용 매핑)

각 섹션마다 **[목적] / [우리가 넣을 내용] / [평가기준 연결] / [주의]** 형식으로 제시한다. 평가기준 코드는 [`평가기준표.md`](./presentation/평가기준표.md) 기준(공통 1-x, 경영 2-x, DS 3-x).

### 국문초록
- **[목적]** 바쁜 심사위원이 이 한 쪽만 봐도 전체를 파악. 배경·목적·방법·주요결과·의의를 간결히.
- **[우리가 넣을 내용]** (문장 흐름 가이드)
  1. *배경/문제*: 사모펀드 사태 이후 자본시장법 개정으로 신탁계약서↔투자제안서(IM) **문서 정합성 검증이 법적 의무화**됐으나, 운용지원팀이 없는 소형운용사는 운용역이 14개 핵심항목을 **수기 대조**한다.
  2. *목적*: LLM으로 검증을 1차 자동화하되, **그 판단을 얼마나 신뢰할 수 있는지를 통계로 증명**.
  3. *방법*: 핵심 4개념(펀드·당사자·보수·환매)을 **온톨로지로 그라운딩**, 결정론 **가드 3종**(형식·출처·제약)으로 출력 검증, **골든셋 30케이스**에 **3조건**(① LLM단독 ② +온톨로지 ③ +가드) 비교.
  4. *결과*: 완성형 **F1 0.815**(LLM 단독 0.762 초과), **재현율 향상·환각률 0**, **McNemar p=0.049(유의)**.
  5. *의의*: 데이터사이언스(neuro-symbolic 검증)와 경영(규제 대응·리스크 관리)의 융합. AI 출력 신뢰성 거버넌스 프레임 제시.
- **[주의]** 숫자는 §4 본문 수치와 **반드시 일치**시킬 것. 초록은 마지막에 쓴다.

### 1. 서론
- **[목적]** 문제의식·연구 필요성·목적·논문 구성을 제시.
- **[우리가 넣을 내용]**
  - **시의성(규제)**: 라임(2019)·옵티머스(2020) → **2021.10.21 시행 개정 자본시장법** → 운용사·판매사·**수탁사 상호 감시·견제 의무** 신설, "운용이 규약·설명서에 부합하는가" 감시 의무화. (출처: [`pestel-규제근거.md`](./presentation/pestel-규제근거.md) §5 1차 출처)
  - **현장 문제**: 검증은 의무가 됐지만 소형사는 여전히 **운용역 수기 대조** → 본업 잠식·함정 누락·책임 부담.
  - **AI 도입의 함정**: LLM 단독은 **환각(없는 조항 생성)·모순 누락**. 그래서 "그냥 읽기"가 아니라 **온톨로지 그라운딩 + 결정론 검증**이 필요.
  - **연구 목적 한 문장**: "신탁계약서 적정성과 IM 반영 일치성을 LLM으로 1차 검증하고, 그 신뢰성을 통계로 증명하는 하네스를 설계·실증한다."
  - **구성 안내**: "본 논문은 이론적 배경(2장), 연구 방법(3장), 분석 결과(4장), 논의(5장), 결론(6장) 순으로 구성된다."
- **[평가기준 연결]** 공통 1-1(문제정의·시의성) 🥇 / 경영 2-3(규제 리스크).
- **[주의]** 대회 주제가 "AI 도입·거버넌스"임을 **명시적으로 연결**("본 연구는 LLM 출력 신뢰성 거버넌스 문제를 다룬다"). niche(B2B 금융)에 머물지 말고 **자본시장 신뢰·투자자 보호**라는 사회적 파급으로 한 단계 확장.

### 2. 이론적 배경 및 선행연구
- **[목적]** 핵심 개념 정의 + 선행연구 한계 + 우리의 차별성.
- **[우리가 넣을 내용]** (예시의 CLV 자리를 우리 개념으로 치환)
  - **개념 정의**: ① LLM 환각(hallucination)과 거버넌스, ② 온톨로지/TBox·ABox, ③ SHACL 제약 검증, ④ **neuro-symbolic**(확률적 LLM + 결정론 규칙 결합), ⑤ 교차검증(cross-check)·정규화·LLM-as-judge.
  - **선행연구 검토**: 계약서 검토 AI, RAG 기반 문서 QA, 온톨로지 기반 컴플라이언스 검증.
  - **선행연구의 한계 → 우리 차별성**:
    - RAG는 "관련 문장 검색"일 뿐 **무엇이 정답 구조인지**를 LLM 머릿속에만 의존 → **누락(빠진 개념)을 못 잡음**. 온톨로지는 필수 개념·규칙을 명시 모델로 박아 **결정론적 진단** 가능. ([`ARCHITECTURE.md`](./ARCHITECTURE.md) §3)
    - 기존 이탈예측류 연구처럼 **"모델 성능"에만 집중**하고 그 출력을 어떻게 신뢰·활용할지 논의가 부족 → 우리는 **신뢰성 자체를 통계로 증명**(McNemar·부트스트랩)하고 비즈니스/규제로 연결.
  - **차별성 핵심 3줄**: 도메인 특화 온톨로지 + 결정론 가드 + **3조건 통계 프레임 + 변조 12유형 골든셋**(이게 진짜 독창 포인트).
- **[평가기준 연결]** 공통 1-3(독창성), DS 3-2(neuro-symbolic 설계).
- **[주의]** "온톨로지+검증"은 학계 선례가 있으므로, **독창성은 "도메인 특화 골든셋 + 3조건 통계 증명"**에 둔다고 명확히 서술. 선행연구는 **APA로 실제 인용**(§참고문헌과 1:1 대응).

### 3. 연구 방법
- **[목적]** 데이터 출처·특성, 변수 구성, 방법론(전처리·분석절차)을 체계적으로.
- **[우리가 넣을 내용]**
  - **데이터**: 펀드 1개(**이지스 블랙ON 일반사모투자신탁제1호**) × 문서 3종 — 신탁계약서(스캔 22p, 기준 문서)·IM(디지털 32p, 변조 주입 대상)·핵심상품설명서(선택). **멀티모달 LLM에 PDF 직접 투입**(별도 OCR 없음). ([`ARCHITECTURE.md`](./ARCHITECTURE.md) §2)
  - **변수/대상 항목**: 핵심 4개념 = 펀드·당사자·보수·환매, 14개 핵심 필드. (예시의 X/Y 자리)
  - **골든셋(정답표)**: 정상 18 + **변조 12 = 30케이스**, 변조 12유형(자릿수·가짜인용·SHACL위반 등). 신탁계약서는 정답 고정, **IM 값만 수동 치환**으로 불일치 생성. 라벨 신뢰성: 2인 독립 채점 후 **Cohen's κ ≥ 0.7**. ([`GOLDENSET.md`](./GOLDENSET.md))
  - **하네스 파이프라인**: `PDF → 추출(LLM) → 가드 3종 → ABox 변환 → SHACL+교차검증 → 정규화/judge → 채점 → 통계`. 6단계 중 **LLM 호출은 추출·judge 2회뿐**, 나머지는 결정론 코드.
  - **가드 3종**: G1 형식(JSON 스키마/재시도) · **G2 출처**(citation 페이지 실재 확인 → 가짜인용 차단) · G3 제약(보수 0~5%·만기>설정일, SHACL 위임).
  - **3조건 설계**(같은 입력·같은 모델, 방법만 다름): ① 베이스라인(LLM 단독) / ② +온톨로지(ABox·SHACL 진단, 수정 안 함) / ③ +가드(G3가 reject/null 집행).
  - **통계기법**: **McNemar**(조건 간 차이의 유의성) + **부트스트랩 신뢰구간**(F1 범위). 채점지표: **재현율 우선** + Precision·F1·Accuracy·환각률.
- **[평가기준 연결]** DS 3-1(데이터 활용) 🥇 / DS 3-2(알고리즘 설계) 🥇.
- **[주의]**
  - **재현율을 왜 우선하는지** 반드시 명시: "정상 다수에 정확도가 높아 보여도 유일한 불일치를 놓치면 컴플라이언스 실패." (리스크 관리 사고 = 경영 2-3 가점)
  - 데이터 한계(단일 펀드)는 여기서 숨기지 말고 §5 한계로 넘긴다고 예고.
  - `database/*.pdf`·`.env`는 **보고서에 원문/스크린샷 게재 금지**(민감 계약서). 케이스는 **가공·익명화**해 제시.

### 4. 분석 결과
- **[목적]** 주요 결과를 **정량적**으로, 시각자료와 함께. 과도한 일반화·인과추론 지양.
- **[우리가 넣을 내용]**
  - **3조건 비교표**(핵심):

    | 지표 | ① non-harness | ② harness | ③ harness+norm |
    |---|:--:|:--:|:--:|
    | F1 | 0.762 | 0.632 | **0.815** |
    | Recall | 0.667 | **1.000** | 0.917 |
    | Precision | 0.889 | 0.462 | 0.733 |
    | 환각률 | 0 | 0 | 0 |

  - **통계**: McNemar ①vs② **p=0.049(유의)**, Bootstrap F1(③) 95% CI **[0.621, 0.952]**.
  - **결정타(정성)**: LLM 단독이 놓친 치명 4건 — **C021 자릿수·C022 날짜·C029 가짜인용·C030 보수한도** — 을 하네스가 전부 차단.
  - **시각자료(필수 생성)**: ① P/R/F1 그룹 막대그래프, ② "놓친 4건 빨강→초록 전환" 표/혼동행렬. (matplotlib; 미생성 상태 → **보고서 마감 전 1순위 작업**)
- **[평가기준 연결]** DS 3-2 / 경영 2-4(성과) / 공통 1-4(표현력).
- **[주의]**
  - **데이터 정직성**: 위 수치는 본 프로젝트 실측값(검증됨). 규제·시장 수치(⚠️ 표시 항목)는 **인용 전 1차 출처 재확인**.
  - 결과 해석은 **데이터 기반 서술**만. "온톨로지가 인과적으로 신뢰를 높인다" 식 인과 단정 금지 → "3조건 비교에서 ~한 차이가 관측됐다"로.
  - ②harness의 F1이 ①보다 낮은(0.632) **역전 현상도 정직하게 보고**하고 §5에서 해석(정밀도 희생·재현율 1.0 달성의 트레이드오프).

  - **결과 해석**: ③ 완성형이 재현율을 유지하며 정밀도를 회복해 F1 최고. ②의 낮은 정밀도는 "탐지는 다 하되 오탐 증가" → 정규화/judge가 이를 교정함을 보임 (neuro-symbolic 역할 분담의 실증).
  - **비즈니스/규제 함의**: 운용역 검토시간↓·놓침 0 보장·온프레미스 보안. 자본시장법상 수탁사·판매사의 문서 정합성 감시 의무에 **1차 자동화 도구로 정합**. 이해관계자 3주체(운용역·준법감시인·경영진) 가치. (출처: [`평가기준표.md`](./presentation/평가기준표.md) 부록 A)
  - **시장 포지셔닝**: 대형사=인력 보유→한계효용 작음 vs **소형사=인력 공백→한계효용 최대**(틈새 진입점).
  - **한계(정직하게)**: ① 단일 펀드(일반화 근거 약함), ② 골든셋 freeze 미완(κ 합의 진행 중), ③ judge 단위 정규화 누락(C021) — G3 결정적 이관으로 개선 예정, ④ 외부 마케팅·계절성 등 미반영. 향후: 다수 펀드·운용사 확장, 비정형 데이터 확장.
- **[평가기준 연결]** 경영 2-1/2-2/2-4 + 2-3(리스크) 🥇 / 공통 1-2.
- **[주의]** **경영 축이 우리 약점**(자기평가 ~28/50)이다. 이 섹션이 **합격 마진의 핵심 레버** — 기술지표를 **비즈니스 언어로 번역**(검토시간↓·오류적발↑·규제리스크↓)하고, 가능하면 SWOT/가치제안 등 **경영 분석도구 1개**를 곁들인다. 동시에 **한계를 솔직히** 적는 것이 성숙도 가점.
