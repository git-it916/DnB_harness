# DnB Harness — 학술제 평가기준 자기평가

> 대상: 제2회 경영 X 데사 연합 학술제 (D&B 2026) **1단계 예선** 평가기준표
> 평가 프로젝트: **DnB Harness** — 온톨로지 기반 사모펀드 신탁계약서 검증 LLM 하네스 (한양대 D&B 7팀)
> 작성: 2026-06-03 · 점수는 **자기평가(추정)** 이며 실제 심사 점수가 아님.

## 0. 평가 구조 이해

심사는 **두 분(경영 교수 / DS 교수)** 이 각각 **공통 50점 + 전공 50점 = 100점** 으로 채점한다. 공통 항목은 두 분이 동일 기준으로 본다.

| 축 | 자기평가 추정 | 한줄 |
|---|:---:|---|
| 공통 (50) | **~40 / 50** | 문제·해법·완성도 강함, 심사용 보고서/시각화는 보강 필요 |
| 경영 전공 (50) | **~28 / 50** (부록 A 반영 시 **~38**) | 약점이었으나 현장 근거로 회복 중 — 소형운용사 타겟·이해관계자맵 확보 |
| 데이터사이언스 전공 (50) | **~43 / 50** | 압도적 강점 — 실제 구현+실측+통계까지 |

**핵심 결론**: 기술(DS)은 예선 수준을 **초과 달성**. 합격·고득점의 레버는 **경영 프레이밍**(이미 있는 기술 자산을 비즈니스/전략/리스크/성과 언어로 재서술) 과 **심사용 보고서·시각자료** 다. *코드를 더 짜는 게 아니라, 가진 것을 더 잘 말하는 것* 이 남은 과제.

## 1. 공통 (50점) — 두 교수 공통

### 1-1. 문제 정의 및 주제 적절성 (15) — 자기평가 **12/15**
**근거 (강):**
- 문제가 명확하고 구체적: "사모펀드 신탁계약서가 제대로 작성됐는지 + 그 내용이 IM(투자제안서)에 정확히 반영됐는지 검증, 그리고 그 판단의 신뢰도를 통계로 증명." (`README.md`, `docs/ARCHITECTURE.md`)
- 시의성: LLM 환각·모순이 금융 컴플라이언스에서 법적 책임으로 직결 → AI 거버넌스 핵심 주제와 정확히 맞물림.
- 실제 도메인 데이터(이지스블랙ON 1호: 계약서 22p + IM 32p) 사용 → 추상론 아님.

**코멘트 (보강):**
- "사회문제" 프레이밍이 niche(B2B 금융)에 머묾 → **사회적 파급**(불완전판매·투자자 피해·자본시장 신뢰)으로 한 단계 확장 서술 권장.
- 대회 주제가 "AI 도입/거버넌스" 임을 **명시적으로 연결**(우리 = "LLM 출력의 신뢰성 거버넌스 프레임워크").

### 1-2. 해결 방안의 종합적 완성도 (15) — 자기평가 **13/15**
**근거 (강):**
- 파이프라인이 논리적·완결적: `PDF → 추출 → 온톨로지 변환 → 가드 3종 → SHACL+교차검증 → 채점 → 통계`.
- 기대효과가 **측정 가능**하게 정의(재현율/정밀도/F1/환각률) 되고 **실측까지 완료**(3조건 비교). 예선에서 보기 드문 완성도.
- 차별화: "온톨로지 그라운딩 + 결정론 가드 + 3조건 통계 증명"의 3중 결합.

**코멘트 (보강):**
- 기대효과를 **비즈니스 임팩트로 환산**한 문장이 약함(정확도 숫자 → "연간 오류 적발 N건/검토시간 X% 단축").
- 기존 사례(상용 계약검토 AI, RAG 기반 QA)와의 **명시적 비교표** 추가 권장.

### 1-3. 창의성 및 독창성 (10) — 자기평가 **8/10**
**근거 (강):**
- "AI 판단을 **얼마나 믿을 수 있는지 통계로 증명**" 이라는 메타 관점이 신선.
- 가짜 인용(G2)·SHACL 범위(G3) 같은 결정론 신호를 LLM 위에 얹는 **하이브리드(neuro-symbolic)** 설계.

**코멘트:**
- 온톨로지+검증 자체는 학계에 선례 존재 → 진짜 독창은 **도메인 특화 골든셋(변조 12유형) + 3조건 통계 프레임**. 이 점을 전면에 내세울 것.

### 1-4. 문서 구성 및 표현력 (10) — 자기평가 **7/10**
**근거 (강):**
- 내부 문서 체계 탁월: `docs/`(ARCHITECTURE·INTERFACES·ROADMAP·STATUS·runbooks), mermaid 온톨로지 다이어그램, 지식그래프 대시보드.

**코멘트 (보강, 중요):**
- 평가 대상은 **제출 보고서**다. 코드·내부문서는 훌륭하나 **심사용 보고서(PDF/슬라이드)로 정제**됐는지가 관건.
- **시각자료 미흡**: 3조건 비교 결과는 강력한 그래프 소재인데 아직 matplotlib 차트 미생성. (P/R/F1 막대, 혼동행렬, McNemar) → 즉시 보강 가능.

**공통 소계: ~40 / 50**

## 2. 경영 전공 (50점) — 상대적 약점 ⚠️

### 2-1. 비즈니스 타당성 및 실현 가능성 (15) — 자기평가 **8/15 → 보강 시 12+/15** (부록 A)
**근거 (강):** 작동하는 PoC로 실현가능성 입증. 로컬 LLM(Ollama)로 **추론 비용 0** → 비용 현실성. **현장 근거 확보**(부록 A): 운용지원팀이 없는 **소형운용사 운용역**이 계약서↔IM을 일일이 수기 대조 → 시간 소모 큼 = 명확한 실수요.
**코멘트 (보강):** 부록 A 의 타겟·도입 시나리오·온프레미스 보안 논거를 보고서 본문에 명시하면 8→12+ 가능. 남은 보강: 정량 ROI 수치(펀드 수×항목×시간).

### 2-2. 경영 전략적 사고 (15) — 자기평가 **6/15 → 보강 시 10+/15** (부록 A)
**근거 (확보):** 부록 A 에 **이해관계자 가치맵**(운용역·준법감시인·경영진)과 **시장 포지셔닝**(인력 공백이 큰 소형운용사에서 한계효용 최대) 정리.
**코멘트 (보강, 최우선):**
- **경영학적 분석도구**(SWOT/밸류체인 등) 최소 1개를 부록 A 위에 얹어 시장·경쟁 분석 완성.
- AI를 "비즈니스 자산"으로 연결: "검증 신뢰성 = 규제 대응·평판 리스크 감소 무형자산" + "온프레미스 = 데이터주권" 논리.

### 2-3. 리스크 및 윤리적 고려 (10) — 자기평가 **7/10**
**근거 (강, 잠재력 큼):** 프로젝트 **자체가 AI 리스크(환각·모순) 대응 도구** → 윤리/리스크 인식이 DNA. "재현율 우선(놓침=사고)" 은 명백한 리스크관리 사고. PDF/PII 커밋금지 등 데이터 보안 인식.
**코멘트:** 다만 **체계적 정리 부족** → AI 도입의 법적(자본시장법)·운영적(하네스 오탐의 책임소재)·윤리적 리스크를 표로 정리하고 각 대응(가드/사람 검토 루프)을 매핑하면 7→9 가능.

### 2-4. 성과 및 임팩트 설계 (10) — 자기평가 **7/10**
**근거 (강):** 측정 가능성 매우 우수 — 정량지표(재현율/정밀도/F1/McNemar p) **실측 산출**.
**코멘트 (보강):** 기술지표를 **비즈니스 KPI로 환산** 부족(검토시간↓, 오류적발↑, 규제리스크↓). 정량/정성 비즈니스 성과로 번역 필요.

**경영 전공 소계: ~28 / 50**

## 3. 데이터사이언스 전공 (50점) — 압도적 강점 🥇

### 3-1. 데이터 활용 계획 타당성 (20) — 자기평가 **15/20**
**근거 (강):**
- 실제 도메인 데이터(신탁계약서·IM) 사용 + **데이터 특성 깊은 이해**: 천분율/약칭/OCR/의미동등/자릿수 함정 등 변조 12유형 분류, 14필드 골든셋 30케이스 설계(`docs/GOLDENSET.md`).
- Multimodal(vision PDF) 입력 처리.

**코멘트 (보강):**
- 데이터 **수집·확장 계획이 좁음** — 펀드 1개(이지스블랙ON1호)뿐. 다수 펀드·운용사로의 일반화 계획, 데이터 양·다양성 부족.
- 골든셋 **freeze 미완**(라벨러 검수·Cohen's κ ≥ 0.7) → 라벨 신뢰성 근거 보강 필요.

### 3-2. 기술적 타당성 및 알고리즘 설계 (30) — 자기평가 **28/30**
**근거 (압도적):**
- 매우 구체적·논리적이며 **실제 구현+작동+실측**: 온톨로지(rdflib/pyshacl SHACL), 가드 3종(G1 형식·G2 출처·G3 제약), 2-pass vision 추출, cross_check, normalize/judge(Claude), 3조건 러너, **통계(McNemar·부트스트랩)**.
- 구현가능성? **이미 구현**됨 — 88개 테스트 통과, 재현 runbook(`docs/runbooks/reproduce-results.md`), 라이브 실행 결과(`database/gemma4_harness/`).
- **실측 결과**: 3조건 F1 ③0.815 > ①0.762 > ②0.632, McNemar ①vs② **p=0.049(유의)**, 환각률 0.

**코멘트:**
- 검증 시스템이라 "추천 알고리즘"류는 아니나, 기준의 "데이터 분석 방법·시스템 설계 구현가능성"을 **예선 기대치 이상으로 충족**.
- 한계 인지·개선안 보유(judge의 단위 정규화 누락 C021 → G3 결정적 이관 계획) → **성숙도 가점**.

**데이터사이언스 전공 소계: ~43 / 50**

## 4. 종합 (ultrathink)

| 심사 | 공통 | 전공 | 총점(추정) |
|---|:---:|:---:|:---:|
| 경영 교수 | ~40/50 | ~28/50 | **~68/100** |
| DS 교수 | ~40/50 | ~43/50 | **~83/100** |

### 4-1. 한 문장 진단
**기술은 예선을 압도하나(특히 DS축), 경영 축에서 "있는 기술을 경영 언어로 번역"하지 못해 점수를 흘리고 있다.** 두 분이 따로 채점하므로, **경영 교수 점수(~68)를 끌어올리는 것이 합격 마진의 핵심 레버**다.

### 4-2. 가장 큰 ROI (코드 X, 서술/시각화 O)
1. **3조건 결과 시각화** (즉시·고효과): P/R/F1 막대그래프 + 혼동행렬 + "LLM 단독이 놓친 치명 4건을 하네스가 차단" 도식. 공통 1-4 + DS 3-2 + 경영 2-4 동시 가점.
2. **경영 프레이밍 1페이지** (최우선): 이해관계자 가치맵 + 도입 시나리오(준법감시 부서) + ROI(검토시간·오류적발·규제리스크) + 리스크 표. → 경영 2-1/2-2/2-4 한 번에 보강.
3. **차별화·기존사례 비교표** (공통 1-2/1-3).
4. **데이터 확장 계획 + 골든셋 freeze 근거(κ)** (DS 3-1).

### 4-3. 솔직한 약점 (심사 전 보완 권장)
- 단일 펀드 데이터(일반화 근거 약함) · 골든셋 freeze 미완(라벨 신뢰성) · 경영 전략도구 부재 · 심사용 보고서/시각자료 미정제 · judge 단위 누락 한계.

### 4-4. 솔직한 강점 (전면에 내세울 것)
- **실제로 작동하고 실측된 3조건 통계 증명**(McNemar p=0.049) — 예선 단계에서 희귀.
- **환각률 0 + 재현율 우선** 이라는 컴플라이언스 정합적 설계 철학.
- **결정론 가드가 LLM이 놓친 치명적 위반(자릿수·가짜인용·보수한도)을 잡는다**는 명확한 가치 증거.

> 점수는 자기평가 추정치다. 실제 심사는 **제출 보고서**로 이뤄지므로, 위 4-2의 보강(특히 시각화 + 경영 프레이밍)이 실점수에 직접적이다.

## 부록 A. 보강된 비즈니스 케이스 (현장 근거 반영)

> 출처: 현장 의견 — "준법감시인과 운용역이 **둘 다** 쓰는 툴이며, **운용지원팀이 없는 소형운용사**에서는 운용역이 계약서↔IM을 **일일이 직접 확인**해야 해 시간 소모가 크다." 이 한 줄이 경영 축(2-1·2-2·2-4)의 빈칸을 메운다.

### A-1. 두 사용자, 하나의 도구

| 사용자 | 현재 고통(As-Is) | 하네스 도입 후(To-Be) |
|---|---|---|
| **운용역** (소형사, 운용지원팀 없음) | 14개 핵심항목 × 2문서(계약서·IM)를 **본인이 직접** 페이지 넘겨 수기 대조 → 본업(운용) 시간 잠식, 사람도 자릿수·가짜인용 함정 누락 | 자동 1차 스크리닝, `needs_review`만 사람 확인. **재현율 1.0 → 안 놓침**. 본업 집중 |
| **준법감시인** | 검토 표준화·증적 부담, 근거 설명 어려움 | `guard_log`·`manifest`·`score.json` = **결정론적 audit trail**. "왜 통과/불통과"를 페이지·규칙으로 제시 → 규제 대응 |
| **경영진** | 컴플라이언스·평판 리스크, 인력 효율 | 리스크↓, 인력 효율↑, **신뢰도를 통계(McNemar)로 증명**해 도구 채택 근거 확보 |

### A-2. 시장 포지셔닝 — 왜 "소형운용사"가 진입점인가

- 대형사는 운용지원팀·준법팀 인력 보유 → AI 한계효용 작음. **소형사는 인력 공백이 커서 한계효용 최대** = 명확한 틈새 타겟.
- 소형사는 전담 검수인력 채용 부담 → **AI 1차 검수가 인력을 대체/보완**.
- **데이터 주권/보안**: 핵심 검증(추출·가드·온톨로지)은 **온프레미스 로컬 LLM(Ollama)** 으로 처리 → 민감한 신탁계약서가 회사 밖으로 안 나감. 클라우드 AI 보안심사·예산 부담이 큰 소형사에 특히 적합. (정규화/judge 는 선택적 클라우드 또는 로컬.)

### A-3. ROI 논리

- **절감 시간 ≈ (펀드 수 × 14항목 × 항목당 수기 대조 시간) − 자동화 처리 시간.** 운용역 시간은 가장 비싼 리소스라 기회비용이 큼.
- **수기 검수 위험의 데이터 증거**: 본 프로젝트 baseline 실험에서 **LLM 단독(=사람과 유사한 눈대중)이 치명 4건(자릿수 함정·날짜·가짜인용·보수한도)을 놓침**. 사람도 같은 함정에 빠짐 → 하네스의 재현율 1.0이 이를 메움. "사람보다 안 놓친다"를 정량 입증.
- **비용**: 로컬 LLM 추론비용 0 + 온프레미스 → 초기 도입 장벽 낮음.
