# DnB Harness — 발표 장표 아웃라인 (1단계 예선)

> 목적: 심사 두 축(경영 교수 / DS 교수) + 공통 기준을 빠짐없이 덮는 **전문가급 PPT** 설계.
> 원칙: **1장 = 1컨셉**. 각 장에 핵심메시지·내용·근거·시각자료·발표멘트·평가기준 연결을 명시.
> 핵심 무기 2개: **PESTEL(규제 중심, P3)** + **가치제안 캔버스 VPC(P12)**.

## 발표 운영 가이드

- **분량**: 핵심 17장 + 부록. (시간 빠듯하면 §맨아래 "12장 압축 매핑" 사용)
- **시간 배분(예: 7분)**: 문제 2분(P2–5) → 솔루션 1.5분(P6–9) → 증명 1.5분(P10–11) → 비즈니스 1.5분(P12–16) → 마무리 0.5분(P17).
- **디자인 원칙(공통 1-4 표현력 직결)**: ① 한 장 한 메시지(제목이 곧 결론인 "메시지 타이틀"), ② 색 3개 이내(메인/강조/회색), ③ 모든 차트 동일 서식, ④ 숫자는 크게·근거는 각주, ⑤ 출처 표기(⚠️ 표시는 실제 출처 확인 필요).
- **데이터 정직성**: 본 프로젝트 실측 수치(F1·McNemar 등)는 검증됨. 규제·시장 수치(⚠️)는 발표 전 1차 출처 확인 필수.

## P1. 표지 — 한 문장으로 각인

- **🎯 컨셉**: 우리가 무엇을, 왜 하는가를 한 줄로.
- **내용**: 제목 「온톨로지 기반 신탁계약서 검증 하네스」 / 부제 "AI의 판단을 *얼마나 믿을 수 있는지*까지 통계로 증명한다" / 팀명(한양대 D&B 7팀)·발표자·날짜.
- **시각자료**: 미니멀 표지. 배경에 흐린 계약서↔IM 대조 이미지 + 체크/실드 아이콘.
- **발표 멘트**: "AI에게 계약서를 그냥 읽히면 없는 조항을 지어내고 모순을 놓칩니다. 저희는 그걸 막고, 막았다는 걸 통계로 증명했습니다."

## P2. Executive Summary — 결론부터 (1장 요약)

- **🎯 컨셉**: 바쁜 심사위원이 이 한 장만 봐도 전부 이해.
- **내용 (4블록)**:
  1. **문제**: 소형운용사 운용역이 신탁계약서↔IM 14항목을 수기 대조 → 시간 소모 + 사람도 함정 누락.
  2. **해법**: 온톨로지 그라운딩 + 결정론 가드 3종 + 정규화/judge로 LLM 출력을 검증.
  3. **증거**: 3조건 실험에서 완성형 **F1 0.815**(LLM 단독 0.762 능가), **재현율↑·환각 0**, 통계적 유의(McNemar p=0.049).
  4. **임팩트**: 운용역 검토시간↓·놓침 0 보장·온프레미스 보안. 규제 강화 흐름에 정합.
- **시각자료**: 4분할 카드(문제/해법/증거/임팩트), 각 카드 핵심 숫자 1개.
- **평가기준**: 공통 1-1·1-2 / 전체 요약.

## P3. 왜 지금인가 — PESTEL (규제 중심) ★

- **🎯 컨셉**: "지금 이 문제를 풀어야 하는" 거시적·규제적 당위.
- **레이아웃**: 6칸 PESTEL, 그중 **L(법규)·P(정치)를 키워 강조**(나머지는 보조).
- **내용**:
  - **L 법규(핵심)** ✅검증: 라임(2019)·옵티머스(2020) 사태 → **2021.10.21 시행 개정 자본시장법**. 운용사·판매사·**수탁사(신탁업자) 상호 감시·견제 의무** 신설 — 수탁사는 운용의 **규약·설명서 부합 여부 감시**, 판매사는 **핵심상품설명서 부합 확인**. → *신탁계약서·IM(설명서) 정합성 검증이 법적 의무화*. (상세·출처: `pestel-규제근거.md`)
  - **P 정치**: 금융당국 투자자보호·감독 강화 기조 지속.
  - **E 경제**: 전문사모운용사 다수·영세화 → 검수 인력난. ⚠️등록사 수(금감원).
  - **S 사회**: 사모펀드 불완전판매로 신뢰 하락 → 투명성·검증 요구↑.
  - **T 기술**: LLM 성숙했으나 **환각 신뢰성**이 도입 걸림돌 → *AI 거버넌스·검증 기술* 수요. (대회 주제 직결)
  - **E 환경**: (약) ESG·공시 신뢰성 맥락.
- **시각자료**: PESTEL 6칸, L·P 칸에 규제 타임라인(사태→개정→현재) 띠 그래프.
- **발표 멘트**: "사모펀드 사태 이후 규제는 '문서 정합성을 누가 검증했는가'를 묻습니다. 그런데 그 검증을 *사람이 수기로* 하고 있습니다."
- **평가기준**: 공통 1-1 시의성 🥇 / 경영 2-3 리스크(규제) 연결.

## P4. 현장의 고통 — 누가, 왜 힘든가 (As-Is)

- **🎯 컨셉**: 추상적 규제 → 구체적 사람의 고통으로 착지.
- **내용**:
  - 사용자: **운용역 + 준법감시인** 둘 다. 특히 **운용지원팀이 없는 소형운용사**.
  - As-Is: 운용역이 신탁계약서(22p)↔IM(32p)의 **14개 핵심 항목**(펀드·당사자·보수·환매)을 **일일이 페이지 넘겨 수기 대조**.
  - 결과: ① 본업(운용) 시간 잠식, ② 사람도 자릿수·가짜인용 함정 누락, ③ 책임 부담.
- **시각자료**: As-Is 플로우(운용역이 두 문서 사이에서 수기 체크하는 일러스트) + "14항목 × 2문서" 강조.
- **발표 멘트**: "대형사엔 운용지원팀이 있지만, 소형사 운용역은 이걸 본인이 다 합니다."
- **평가기준**: 경영 2-1 비즈니스 타당성(실수요) / 공통 1-1.

## P5. 문제의 본질 — AI에 그냥 맡기면 안 되는 이유

- **🎯 컨셉**: "그럼 LLM에 시키면 되지 않나?"에 대한 반박 = 우리 존재 이유.
- **내용**:
  - LLM 단독은 **환각**(없는 조항 생성)과 **모순 누락**을 일으킴.
  - 사람도 LLM도 같은 함정: **자릿수(0.3 vs 0.03), 가짜 인용(999p), 보수 한도 초과, 날짜 추론**.
  - → "그냥 읽기"가 아니라 **개념 지도(온톨로지)로 그라운딩 + 결정론 검증**이 필요.
- **시각자료**: 함정 4종 아이콘 + "AI도 사람처럼 속는다" 카피. (P11에서 데이터로 증명 예고)
- **발표 멘트**: "이건 P11에서 데이터로 증명합니다 — LLM 단독이 가장 위험한 4건을 통과시켰습니다."
- **평가기준**: 공통 1-1 문제정의 / 공통 1-3 창의성(접근의 당위).

## P6. 솔루션 개요 — 온톨로지 그라운딩 + 하네스

- **🎯 컨셉**: 우리 해법의 전체 그림 한 장(파이프라인).
- **내용**: 핵심 3축 — ① **온톨로지**(4개념을 정해진 틀로 박아 누락·모순 결정론 탐지), ② **가드 3종**(형식·출처·제약), ③ **3조건 통계 증명**.
  - 파이프라인: `PDF → 추출 → 온톨로지 변환(ABox) → 가드 3종 → SHACL+교차검증 → 정규화/judge → 채점 → 통계`.
- **시각자료**: 좌→우 파이프라인 다이어그램(각 단계 아이콘). 온톨로지/가드/통계 3축 색 구분.
- **발표 멘트**: "그라운딩으로 잡고, 가드로 거르고, 통계로 증명합니다."
- **평가기준**: 공통 1-2 해결방안 완성도 / DS 3-2 시스템 설계.

## P7. 기술 ① 온톨로지 & SHACL — 결정론적 그라운딩

- **🎯 컨셉**: "개념 지도"가 어떻게 누락·모순을 *결정론적으로* 잡나.
- **내용**:
  - 4개념(펀드·당사자·보수·환매) TBox + 추출 결과를 RDF ABox로 변환(rdflib).
  - SHACL(pyshacl)로 비즈니스 제약 검증: 예) 보수 0~5%, 만기일 > 설정일, 신탁업자 1명.
  - 확률(LLM)이 아니라 **규칙 위반을 결정론적으로** 산출 → 재현·설명 가능.
- **시각자료**: 온톨로지 그래프(mermaid, 4개념 노드+관계) + SHACL 위반 예시 캡처.
- **발표 멘트**: "여긴 모델의 '감'이 아니라 규칙입니다. 같은 입력이면 같은 판정."
- **평가기준**: DS 3-2 기술 타당성 🥇 / 공통 1-3 창의성(neuro-symbolic).

## P8. 기술 ② 가드 3종 — 형식·출처·제약

- **🎯 컨셉**: LLM 출력을 거르는 3겹 결정론 필터.
- **내용**:
  - **G1 형식**: JSON 스키마 위반 차단(+재시도).
  - **G2 출처**: citation 페이지가 실제 PDF 범위 안인지 → **가짜 인용(999p) 차단**.
  - **G3 제약**: 보수 범위·만기>설정 + SHACL 위임 → **보수 한도 초과·논리 위반 차단**.
  - reject 시 해당 값 null화(불변 보존).
- **시각자료**: G1→G2→G3 3단 필터 + 각 가드가 잡는 함정 매핑.
- **발표 멘트**: "가짜 페이지 인용과 한도 초과 보수는 LLM이 못 잡지만 G2·G3는 결정론으로 잡습니다."
- **평가기준**: DS 3-2 / 경영 2-3(리스크 차단).

## P9. 기술 ③ 정규화·judge·채점·통계 — 하이브리드 완성

- **🎯 컨셉**: 결정론으로 못 푸는 "표기·의미 차이"는 LLM로, 그리고 전부 점수화.
- **내용**:
  - **정규화(Claude)**: "1,000분의 8.9" ≡ "0.89%" 같은 단위·표기 통일.
  - **judge(Claude)**: 의미 동등("환매 불가" ≡ "폐쇄형") 판정.
  - **채점**: gold_label↔판정 매핑으로 P/R/F1/환각률 산출(LLM 불필요 순수 함수).
  - **통계**: McNemar(조건 간 유의성) + 부트스트랩(F1 신뢰구간).
- **시각자료**: 결정론(가드) ↔ 확률(정규화/judge) 역할 분담 도식 + 채점→통계 흐름.
- **발표 멘트**: "결정론과 LLM을 역할 분담시키고, 결과를 숫자로 못박았습니다."
- **평가기준**: DS 3-2 알고리즘 설계 / 경영 2-4 측정가능성.

## P10. 검증 설계 — 3조건 실험 + 골든셋

- **🎯 컨셉**: "효과가 있다"를 *공정하게* 측정하는 실험 설계.
- **내용**:
  - **골든셋 30케이스**: 변조 12 + 정상 18, 14개 항목, 변조 12유형(자릿수·가짜인용·SHACL위반 등), 난이도·신호 라벨. (정답표)
  - **3조건**(같은 입력, 방법만 다름): ① non-harness(LLM 단독) / ② harness(가드) / ③ harness+norm(가드+정규화+judge).
  - **지표**: 재현율 우선(놓치면 컴플라이언스 사고) → F1로 균형 확인.
- **시각자료**: 3조건 비교 실험 다이어그램 + 골든셋 구성 미니 표.
- **발표 멘트**: "같은 30문제를 세 방식으로 풀게 하고 채점했습니다."
- **평가기준**: DS 3-1 데이터 활용 🥇 / DS 3-2 방법론 / 경영 2-4.

## P11. 검증 결과 — 숫자로 증명 (핵심 장표 ★)

- **🎯 컨셉**: 우리 해법이 LLM 단독을 이긴다 + 위험한 걸 잡는다.
- **내용 (3조건 표)**:
  | | ① non-harness | ② harness | ③ harness+norm |
  |---|:--:|:--:|:--:|
  | F1 | 0.762 | 0.632 | **0.815** |
  | Recall | 0.667 | **1.000** | 0.917 |
  | Precision | 0.889 | 0.462 | 0.733 |
  | 환각률 | 0 | 0 | 0 |
  - **McNemar ①vs② p=0.049(유의)**, Bootstrap F1(③) 95% CI [0.621, 0.952].
  - **결정타**: LLM 단독이 놓친 치명 4건(**C021 자릿수·C022 날짜·C029 가짜인용·C030 보수한도**)을 하네스가 전부 차단.
- **시각자료**: ① P/R/F1 그룹 막대그래프, ② "놓친 4건" 빨강→초록 전환 표. (⚠️ matplotlib로 생성 필요)
- **발표 멘트**: "완성형이 F1 최고로 LLM 단독을 이겼고, 무엇보다 *가장 위험한 4건*을 사람·LLM과 달리 놓치지 않았습니다."
- **평가기준**: DS 3-2 / 경영 2-4 성과 / 공통 1-2.

## P12. 가치제안 캔버스 (VPC) ★

- **🎯 컨셉**: 고객의 고통과 우리 가치가 *정확히* 맞물림을 한 장으로.
- **레이아웃**: 표준 VPC — 오른쪽 원(고객 프로필) ↔ 왼쪽 사각(가치 맵).
- **내용**:
  - **고객 프로필(운용역 중심)**:
    - *Jobs*: 계약서↔IM 정합성 확인, 규제 준수 확신, 펀드 설정 전 문서 검수.
    - *Pains*: 14항목 수기 대조 시간, 본업 잠식, 자릿수·가짜인용 누락 위험, 책임 부담, 운용지원팀 부재.
    - *Gains*: 빠른 검수, "안 놓쳤다"는 확신, 감사 증적, 본업 집중.
  - **가치 맵(하네스)**:
    - *Products*: 검증 하네스(추출+가드+온톨로지+통계).
    - *Pain Relievers*: 자동 1차 스크리닝(needs_review만 사람), **재현율 1.0=안 놓침**, **온프레미스=유출 방지**, audit trail=책임 근거.
    - *Gain Creators*: 검토시간↓, **신뢰도 통계 증명**, 검토 표준화.
- **시각자료**: 정식 VPC 도형. Pain↔Reliever, Gain↔Creator를 선으로 연결(fit 강조).
- **발표 멘트**: "운용역의 고통 하나하나에 우리 기능이 1:1로 대응합니다 — 이게 product–market fit입니다."
- **평가기준**: 경영 2-2 전략·이해관계자 🥇 / 경영 2-1.

## P13. 이해관계자 & 시장 — 왜 소형운용사인가

- **🎯 컨셉**: 한 도구가 3주체에게 가치 + 진입 시장이 명확.
- **내용**:
  - **이해관계자 가치맵**: 운용역(시간·본업) / 준법감시인(표준·증적) / 경영진(리스크·인력효율·신뢰 통계).
  - **시장 포지셔닝**: 대형사=인력 보유→한계효용 작음 vs **소형사=인력 공백→한계효용 최대**(틈새 진입).
  - **온프레미스 데이터주권**: 민감 계약서를 로컬 LLM으로 처리→외부 유출 X(클라우드 보안심사 부담 큰 소형사에 적합).
  - (선택) **TAM/SAM/SOM**: 전체 운용사 → 전문사모운용사 → 운용지원팀 없는 극소형사. ⚠️수치 출처 확인.
- **시각자료**: 이해관계자 3원 가치맵 + 시장 깔때기(TAM/SAM/SOM).
- **발표 멘트**: "인력 공백이 가장 큰 소형사에서 우리 도구의 효용이 가장 큽니다."
- **평가기준**: 경영 2-1 타당성 / 경영 2-2 전략.

## P14. 리스크 & AI 거버넌스 — 리스크 매트릭스

- **🎯 컨셉**: AI 도입 리스크를 인식하고 *각각 완화책을 설계*했다.
- **내용 (발생가능성×영향 매트릭스 + 완화책)**:
  | 리스크 | 발생 | 영향 | 완화책 |
  |---|:--:|:--:|---|
  | AI 환각으로 오류 승인 | 중 | 高 | 가드 + 재현율 1.0 |
  | 데이터 유출(민감 계약서) | 저 | 高 | 온프레미스 로컬 LLM |
  | 책임소재 모호 | 중 | 中 | audit trail(guard_log/manifest) |
  | 하네스 오탐 과다 | 중 | 中 | 정규화/judge로 정밀도↑ |
  | 규제 미준수 | 저 | 高 | SHACL 비즈니스 규칙 집행 |
  - 운영 원칙: **사람-검토 루프**(하네스는 1차 스크리닝, 최종 판단은 사람) → 책임·윤리 안전장치.
- **시각자료**: 2×2(또는 3×3) 리스크 히트맵 + 완화 화살표.
- **발표 멘트**: "AI를 신뢰의 문제로 보고, 리스크마다 결정론적 안전장치를 뒀습니다."
- **평가기준**: 경영 2-3 리스크·윤리 🥇.

## P15. 성과 & ROI — 측정 가능한 임팩트

- **🎯 컨셉**: 효과를 비즈니스 KPI로 환산.
- **내용**:
  - **정량 KPI**: 검토시간 절감 = (펀드 수 × 14항목 × 항목당 수기시간) − 자동화 시간 ⚠️현장 수치 필요 / 오류 적발 건수 / **재현율 1.0=누락 0** / 환각률 0.
  - **정성**: 검토 표준화, 규제 대응력, 운용역 본업 집중.
  - **비용**: 로컬 LLM 추론비용 0 + 온프레미스 → 도입 장벽 낮음.
  - **측정틀(로직모델)**: input(문서) → output(검증 리포트) → outcome(검토시간↓·놓침↓) → impact(컴플라이언스 리스크↓).
- **시각자료**: ROI 막대(As-Is 수기시간 vs To-Be) + KPI 트리.
- **발표 멘트**: "성과를 '느낌'이 아니라 시간·건수·재현율로 측정합니다."
- **평가기준**: 경영 2-4 성과·임팩트 🥇 / 경영 2-1 비용.

## P16. 차별화 — 경쟁 포지셔닝 2×2

- **🎯 컨셉**: 기존 사례와 다른 빈 사분면을 우리가 점유.
- **내용**:
  - 축: X=도메인 특화도(범용↔신탁 특화), Y=신뢰성 보장(확률적↔**결정론+통계증명**).
  - 배치: 수기 검토 / 범용 LLM·RAG QA / 상용 legaltech / **우리=우상단(특화×결정론+통계)**.
  - 차별점 3줄: 도메인 온톨로지, 결정론 가드, 통계적 신뢰 증명.
- **시각자료**: 2×2 산점도, 우상단에 우리 로고.
- **발표 멘트**: "비어 있는 사분면 — 도메인 특화이면서 신뢰성을 통계로 증명하는 자리 — 를 우리가 차지합니다."
- **평가기준**: 공통 1-2 차별화 / 공통 1-3 독창성.

## P17. 로드맵 & 클로징

- **🎯 컨셉**: 지금까지 한 것(실증) → 다음 것 → 한 문장 마무리.
- **내용**:
  - **현재(완료)**: 추출·온톨로지·가드·정규화·통계 구현, 3조건 실증, 88 테스트, 재현 가능.
  - **다음**: 골든셋 freeze(라벨 κ 합의), 보수 단위 G3 결정적 이관(재현율 1.0 회복), 다수 펀드·운용사 확장, 현장 ROI 수치화.
  - **클로징 한 줄**: "AI가 신탁계약서를 *대신 읽는 게 아니라*, 사람이 믿고 쓸 수 있게 *검증*합니다."
- **시각자료**: 타임라인(완료=실선, 예정=점선) + 마지막 슬로건.
- **평가기준**: 공통 1-2 완성도 / 경영 2-1 실현가능성.
