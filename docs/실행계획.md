# 실행 계획 (EXECUTION PLAN) — MVP 버전

> **목표**: 잔가지 다 쳐내고 **통계값 하나를 뽑아내는 최소 하네스(MVP)**.
> **유지**: 온톨로지 + 하네스(가드) 엔지니어링. **골든셋 30개. 펀드 1개(이지스).**
> **기간**: 4주 (+1주 버퍼). 설계 배경은 [`PLAN.md`](./PLAN.md), 이 문서는 실행만.

---
## 0. MVP 한 장 요약

- **만드는 것**: 신탁계약서·IM(투자제안서)에서 핵심 값을 **온톨로지(개념 지도)** 에 맞춰 뽑고, ① 계약서가 규칙을 지켰는지(SHACL) ② 그 값이 IM과 일치하는지(교차검증) 를 자동 판정하는 하네스.
- **증명하는 것(통계값)**: 3단계로 갈수록 **불일치 탐지가 좋아지는가?**
  `① LLM 단독 → ② +온톨로지 추출 → ③ +가드(풀 하네스)` → 골든셋 30문항에 **McNemar 검정 + 부트스트랩 신뢰구간**.
- **성공 기준**: "③이 ①보다 통계적으로 유의하게 낫다"는 결과 + 그래프 1장 + 발표.

## 1. 스코프 동결 (다시 안 부풀리기 위한 선)

| ✅ 남긴다 (트렁크) | ❌ 잘랐다 (잔가지) |
|--------------------|--------------------|
| 온톨로지 **코어 4개념**(펀드/당사자/보수/환매) | 8개념·운용·분배·해지·위험 개념 |
| SHACL 규칙 검증 | 복잡한 추론기(owlready 등) |
| 가드 **3종**(형식·출처·제약) | 가드 5종 → 3종, 심각도 보정 가드 |
| 교차검증(계약서값 vs IM값) = 핵심 태스크 | RAG 베이스라인 **인프라** |
| 골든셋 **30문항 1개 셋** | 3층 방대 골든셋·여러 펀드 |
| 변조 = **IM 값 수동 치환** | PDF 자동 변조 툴(pymupdf 오버레이) |
| McNemar + 부트스트랩 | t-test·회귀·다중검정 보정(여유 되면) |

> 한 줄: **온톨로지·가드는 살리고, "더 있으면 좋은 것"은 전부 뒤로.**

---

## 2. 팀 (5명, 최소 분담)

| 이름 | 한 줄 역할 |
|------|-----------|
| **승훈** (PM) | 골든셋 최종 확정·통계(McNemar·Bootstrap)·보고·일정 |
| **종현** | 온톨로지(`trust_fund.ttl`/`shapes.ttl`)·LLM 추출·교차검증 로직 |
| **건** | 가드 3종·CLI·변조 적용·3조건 실험 실행 |
| **승연** | 검증(채점기·메트릭·환각률)·3조건 비교 리포트 |
| **리나** | 골든셋 검수(라벨러)·κ 합의·발표 장표 |

> 운영 룰은 딱 둘: **(1) main 직접 push 금지, PR로.** **(2) 막히면 24h 안에 PM 호출.**

## 3. MVP 하네스 (이게 전부)

```
PDF 3종
  → [추출]   AI가 4개념 값을 정해진 빈칸(JSON)에 뽑음
  → [변환]   값을 개념 지도에 끼움 (ABox)
  → [가드]   3종 검사: 형식 / 출처(없는 페이지 인용 차단) / 제약(말도 안 되는 값 차단)
  → [판정]   SHACL: 계약서가 규칙 지켰나  +  교차검증: 계약서값 = IM값 ?
  → [채점]   골든셋 30문항과 비교 → 점수
  → 리포트
```

**통계값을 뽑는 3조건** (같은 30문항, 같은 모델·seed):

| 조건 | 무엇 | 온톨로지 | 가드 |
|------|------|:---:|:---:|
| ① 베이스라인 | "계약서와 IM에서 안 맞는 값 찾아줘"를 자유 질문 | ✗ | ✗ |
| ② +온톨로지 | 4개념 구조화 추출 후 값 자동 비교 | ✓ | ✗ |
| ③ +가드(풀) | ②에 가드 3종 추가 | ✓ | ✓ |

→ ①→②→③ 갈수록 불일치 탐지(재현율·F1)가 오르는지 측정. **이 차이가 우리 결과.**

## 4. 골든셋 30개 만드는 법

**마스터 시트 1장**(`golden_master.csv`) — 컬럼만 채우면 됨:

| 개념 | 계약서 값 | 계약서 쪽 | IM 값 | IM 쪽 | 정답라벨 |
|------|-----------|-----------|-------|-------|----------|
| 운용보수 | 0.7 | 12 | 0.7 | 8 | 일치 |
| 운용보수(변조) | 0.7 | 12 | 0.5 | 8 | **불일치** |

- **구성**: 정상 18문항 + **변조 12문항**(불일치를 일부러 주입 → "AI가 잡나" 측정용).
- **변조 방법(최소)**: **신탁계약서는 정답 기준으로 고정**, **IM 값만 손으로 바꿔서** 불일치 생성. (스캔본 편집 안 함)
- **신뢰성(κ)**: **리나 + 승훈이 같은 30문항을 따로 채점** → 일치도(κ) 계산, 0.7 넘으면 OK. 다른 항목만 토론 후 PM이 최종 확정.
- **파생**: 시트 freeze 후 스크립트로 채점용 파일 생성(승연/종현).

## 5. 4주 계획 (팀원별, 최소)

### W1 — 토대: 온톨로지 4개념 + 추출 + 골든셋 시작
- **승훈**: 골든셋 시트 양식·라벨링 가이드(`docs/golden_master.md` 완료) + `docs/INTERFACES.md`(manifest·score 스키마) + 통계 모듈 골격(`src/stats/`).
- **종현**: `trust_fund.ttl`(4개념) + 추출 프롬프트(빈칸 강제) + 추출→JSON.
- **건**: AI 호출 공용 함수(재시도·사용량) + 출처 검사 가드 초안.
- **승연**: `score.json` v0 스키마 확정 + 채점기 코어(Accuracy·Precision·Recall·F1) 순수함수 + mock 데이터 단위테스트. (빠른 테스트 사이클 확보)
- **리나**: `tests/golden/golden_master.csv` 30개 PDF 대조 검수 시작 + 합성 케이스(C002·C014·C029) 의견. 가이드: `docs/golden_master.md`.
- ✅ 끝 신호: 명령어 한 줄로 PDF→AI 답이 뜬다.

### W2 — MVH: 끝→끝 + 골든셋 30 완성
- **승훈**: κ 계산 + 골든셋 정답 확정(freeze) + 통계 모듈 구현(McNemar 골격).
- **종현**: JSON→개념 지도(ABox) + `shapes.ttl`(4개념 규칙) + **교차검증**(계약서값 vs IM값 비교).
- **건**: 형식·출처 가드 완성 + 켜고/끄기 스위치.
- **승연**: 환각률 + 세분화 지표(필드·난이도·변조유형·harness_signal별) 추가 → 종현의 cross_check 결과로 **첫 진짜 `score.json` 산출**.
- **리나**: 검수 완료 → `labeler_v1_rina.csv` 제출 → PM과 κ 측정 → 골든셋 30 freeze.
- ✅ 끝 신호: PDF 넣으면 **점수 리포트 파일**이 나온다 / 골든셋 30 freeze + κ≥0.7.

### W3 — 가드 + 변조 + 3조건 실험
- **승훈**: 3조건 정의·골든셋 freeze·실행 감독.
- **종현**: ① 베이스라인(자유 질문) 모드 구현 + 추출 안정화.
- **건**: 제약 가드 추가 + **변조 적용**(리나 표대로 IM 값 치환) + **3조건 전수 실행** → 결과 저장.
- **승연**: 3개 `score.json` → `compare.md` 생성기 + 첫 ③ vs ① 비교 결과 도출.
- **리나**: 장표 뼈대(문제·방법·결과 틀) + 변조 케이스 사례 정리(발표용).
- ✅ 끝 신호: 3조건 결과가 한 번에 나오고, "AI가 불일치를 잡았나"가 숫자로 찍힌다.

### W4 — 통계 + 발표
- **승훈**: **McNemar(③ vs ①)** + 결과 해석 + 보고서.
- **종현**: 그래프(3조건 F1 막대 + 비용 1장).
- **건**: **부트스트랩 신뢰구간** + "온톨로지·가드가 효과 있다" 결론 1줄.
- **승연**: 발표용 최종 비교 표·그래프 보조 + `compare.md` 다듬기.
- **리나**: 장표 완성 + 발표 대본.
- ✅ 끝 신호: 통계표 + 그래프 1~2장 + 발표 리허설.
