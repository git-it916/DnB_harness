무엇이 달라졌나

핵심 차이는 LLM에게 바로 “같냐/다르냐”를 맡기지 않고, 온톨로지 필드 정책을 먼저 실행한 뒤 필요한 필드만 제한된 judge로 넘기는 것이다.

기존 harness+정규화

정규화와 일반 judge가 많은 의미동등 판단을 담당했다.

기존
  • Claude normalization이 숫자·단위·표현 차이를 한 번에 정규화했다.
  • 정규화 이후에도 애매하면 generic same/different judge가 최종 판단했다.
  • C021 판매보수 0.3% vs 0.03% 같은 단위 함정을 judge가 “동일”로 오판했다.
  • 왜 맞았는지/틀렸는지 필드별 정책 증거가 부족했다.

새 ontology_policy_judge

온톨로지 정책이 먼저 비교하고, LLM은 제한된 필드 추론만 한다.

신규
  • 명시 단위, 날짜, 기간, 부재 의미를 결정론 canonical comparison으로 먼저 확정한다.
  • LLM judge는 non-decisive이고 policy가 허용한 필드에만 호출된다.
  • 환매가능여부·만기일·환매수수료는 필드 전용 classifier가 구조화하고 코드가 최종 판정한다.
  • needs_review는 불일치 확정이 아니라 review로 분리했다.

전체 온톨로지 기반 구현안

온톨로지는 “무엇을 비교해야 하는가”를 고정하고, field policy는 “각 필드를 어떻게 비교할 것인가”를 실행 가능하게 만든다.

01 · Extract

LLM 추출은 14개 필드를 채운다

Fund, Party, FeeSchedule, RedemptionTerms의 계약서/IM 값을 ExtractionResult로 구조화한다.

Pydantic schema
02 · Guard

G1/G2/G3가 형식·출처·제약을 결정론적으로 검사한다

가짜 페이지, 보수 범위 초과, 형식 오류는 LLM 없이 reject/null 처리한다.

deterministic
03 · Ontology

ABox와 SHACL이 구조와 규칙을 진단한다

trust_fund.ttl과 shapes.ttl이 필수 개념·범위·관계를 검증한다.

TBox · ABox · SHACL
04 · Policy

field_policies.yaml이 필드별 비교 방식을 결정한다

percent, date, duration, boolean, absence semantics와 judge fallback 허용 여부를 14개 필드 전체에 명시한다.

14 field policies
05 · Canonical

결정론 canonical comparison이 먼저 결론을 낸다

예: “연 1,000분의 8.9”와 “0.89%”는 같은 percent로, “0.3%”와 “0.03%”는 다른 값으로 확정한다.

no LLM call
06 · Field Judge

남은 필드만 제한된 LLM classifier로 보낸다

환매 가능 여부, 만기일, 환매수수료는 각각 전용 enum/날짜로 구조화하고 최종 판정은 코드가 한다.

structured fallback
07 · Score

match / mismatch / review를 분리해 채점한다

needs_review는 mismatch로 뭉개지 않고 “모르겠다”로 집계한다.

review count

이번에 수정한 것

작업의 핵심은 “LLM을 더 많이 쓰는 것”이 아니라, LLM이 맡는 일을 필드별로 좁히고 코드가 최종 판정을 통제하도록 바꾼 것이다.

01

실행 가능한 field policy 추가

14개 추출 필드 전체를 ontology/field_policies.yaml로 등록했다. 각 필드가 어떤 타입인지, 어떤 canonicalizer를 쓰는지, judge fallback을 허용하는지 명시했다.

02

canonical 패키지 추가

src/canonical/에서 percent/date/duration/boolean/absence를 결정론적으로 파싱하고 CrossCheckResult에 canonical evidence를 남긴다.

03

review 분리 채점

needs_review를 mismatch로 계산하지 않고 predicted_label=review로 분리했다. score.json에는 review count/rate가 추가된다.

04

환매 가능 여부 classifier

LLM이 양쪽을 yes/no/conditional/unknown으로 독립 분류하고, 코드는 yes/no 조합만 자동 확정한다.

05

만기일 classifier

추출된 설정일을 support context로 사용해 절대일자와 “2년” 같은 기간 표현을 ISO 만기일로 비교한다.

06

환매수수료 classifier

양쪽을 같은 fee status enum으로 분류한다. no_fee와 not_applicable_no_redemption은 redemption_fee 필드에서 같은 리스크 상태로 본다.

필드 전용 judge의 방식

LLM은 “최종 판정자”가 아니라 “추출값을 필드별 비교 단위로 변환하는 분류기”가 된다.

  • 환매 가능 여부: yes/no/conditional/unknown으로 분류한다.
  • 만기일: 절대일자와 설정일 기준 기간을 ISO 날짜로 분류한다.
  • 환매수수료: fee_specified/no_fee/not_applicable/conditional/unknown으로 분류한다.
  • conditional, unknown, 낮은 confidence는 자동 판정하지 않고 review로 남긴다.

다른 사람에게 설명할 핵심 메시지

이 작업은 “LLM judge를 더 믿자”가 아니라 “온톨로지 정책으로 LLM judge를 제한하자”에 가깝다.

  • 온톨로지는 비교할 도메인 필드와 제약을 고정한다.
  • field policy는 온톨로지 개념을 실행 가능한 비교 정책으로 바꾼다.
  • canonical comparison은 단위·날짜·부재 의미를 LLM 없이 먼저 확정한다.
  • LLM fallback은 필드별 구조화 classifier로 제한한다.
  • 현재 남은 리스크는 C002 펀드명 약칭 1건이다.
