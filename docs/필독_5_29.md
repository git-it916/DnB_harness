## 4. 추출 파이프라인 (PM 전담, 동작 확인)

### 4.1 모듈 (총 ~480줄)

| 파일 | 역할 | 줄수 |
|---|---|--:|
| [`src/ingest/pdf_to_text.py`](../src/ingest/pdf_to_text.py) | 디지털/스캔 자동 분기. 디지털=pdfplumber, 스캔=Gemma 4 vision (base64 PNG). Poppler 자동 탐색. | ~140 |
| [`src/client/ollama_client.py`](../src/client/ollama_client.py) | Ollama HTTP + JSON Schema 강제 + vision images + model_digest 로깅. | ~130 |
| [`src/extraction/side_schemas.py`](../src/extraction/side_schemas.py) | `FundSide`·`PartySide`·`FeeScheduleSide`·`RedemptionTermsSide` + `SideExtraction` + `merge_sides(contract, im) → ExtractionResult`. | ~110 |
| [`src/extraction/extractor.py`](../src/extraction/extractor.py) | **2-pass orchestration**: contract vision pass + IM text pass → merge. 프롬프트 인라인. | ~150 |

### 4.2 흐름
```
PDF 2개
   │
   ▼
ingest_pdf(contract.pdf)  → 22개 base64 PNG (vision)
ingest_pdf(im.pdf)        → 32 페이지 텍스트 (digital)
   │
   ▼
Pass 1: OllamaClient.generate(contract images + SideExtraction 스키마)
Pass 2: OllamaClient.generate(IM 텍스트 + SideExtraction 스키마)
   │
   ▼
merge_sides(contract_side, im_side) → ExtractionResult
```

### 4.3 왜 vision 모드인가 (Tesseract 안 씀)
- 계약서가 **스캔본 (text=0)** → pdfplumber 무용
- 원래 plan 은 Tesseract OCR 였으나, Gemma 4 가 vision 지원 → **시스템 설치 부담 회피 + 단일 모델로 통일**
- DPI=200 으로 한국어 OCR 충분 (§6.3 검증)

### 4.4 백엔드 추상화 (LLMBackend Protocol) — **YAGNI 결정**
INTERFACES.md 에 시그니처만 박아두고 구현 안 함. Claude vs Gemma 비교 실험 필요해지면 그때 추가. **5회 테스트는 단일 백엔드 충분**.

## 6. 검증된 실험 (D+0)

### 6.1 V1 — 첫 5회 추출 baseline
- 위치: `database/gemma4/run_NN_seedM/{extraction.json, guard_log.json, meta.json}`
- 설정: `temp=0.1`, `DPI=150`, seeds [42, 1, 2, 3, 4]
- 결과: **5/5 성공**, 평균 wall 174s, eval 106s, 가드 거절 0
- 문제: 이지스블**럭**/블루 (vision OCR 오류), 일부 필드 결측

### 6.2 (ii) temp=0 안전성 smoke
- [`scripts/smoke_temp0.py`](../scripts/smoke_temp0.py)
- text-only prompt × temp ∈ {0.0, 0.05, 0.1} × 2회 = 6 호출
- 결과: **6/6 동일 출력**, degeneration 없음
- → PLAN §4.4의 "Q4 폭주" 우려는 text-only 한정 false alarm

### 6.3 (iii) V2 — DPI 200 + temp=0
- [`scripts/extract_v2.py`](../scripts/extract_v2.py)
- 설정: temp=0, DPI=200, seed=42 × 2회 (run_A, run_B)
- 위치: `database/gemma4_v2_t0_d200/run_{A,B}/`
- **결과**: 이지스블**럭** → 이지스블**랙** ✅ + 4 필드 (inception/sales_fee/is_redeem 등) 결측 해소
- **결정론**: 8/9 필드 일치 (sales_fee 만 run_A 채워짐, run_B null). Vision 입력은 temp=0 이어도 100% 결정론 X — 통계 단에서 다중 실행 처리

### 6.4 (iv) G2 가드 골든셋 C029 시뮬
- [`scripts/test_g2_fake_citation.py`](../scripts/test_g2_fake_citation.py)
- 변조 2종: (a) page=999, (b) document mismatch
- **결과**: 두 변조 모두 catch + null화 ✅
- 총 16 G2 이벤트 (pass 14, reject 2) — LLM 호출 0회 (순수 결정론)

### 6.5 (v) 최종 채택 → 코드 반영
- [`src/ingest/pdf_to_text.py`](../src/ingest/pdf_to_text.py): `IMAGE_DPI = 200`
