# 2. 이론적 배경 및 선행연구

## 2.1 하네스와 온톨로지 기반 검증

본 연구에서 **하네스(harness)**는 신탁계약서와 투자제안서(IM)의 정합성을 검증하기 위해 LLM 기반 정보 추출, 형식·출처·제약 검사, 온톨로지 변환, 문서 간 교차비교, 선택적 의미 판정 및 성능평가를 하나의 절차로 연결한 통합 검증 파이프라인을 의미한다. 하네스의 목적은 LLM의 응답을 곧바로 최종 결과로 사용하는 것이 아니라, 각 단계의 입력·출력과 판정 근거를 통제하여 결과를 재현하고 오류 원인을 추적할 수 있게 하는 데 있다.

온톨로지는 특정 도메인의 개념, 속성, 관계 및 규칙을 명시적으로 표현한 지식 구조이다. 본 연구는 신탁계약서의 핵심 정보를 `Fund`, `Party`, `FeeSchedule`, `RedemptionTerms`의 네 개 개념으로 구조화하였다. TBox는 이들 개념과 관계를 정의한 스키마이며, ABox는 LLM이 계약서와 IM에서 추출한 실제 값을 RDF 인스턴스로 표현한 데이터이다. 두 문서의 정보는 각각 독립된 ABox 인스턴스로 생성되지만 동일한 개념 구조를 따르기 때문에 문서별 표현 차이와 관계없이 같은 기준에서 진단할 수 있다.

ABox의 구조와 규칙 검증에는 W3C 표준인 SHACL(Shapes Constraint Language)을 사용하였다. SHACL은 RDF 그래프에 필요한 속성의 존재 여부, 값의 자료형과 개수, 수치 범위 및 노드 간 관계를 명시하고 위반 리포트를 생성한다(W3C, 2017). 본 연구에서는 펀드명의 존재, `Fund`와 당사자·보수·환매조건의 연결, 계약서 내 운용사와 신탁업자의 존재, 신탁업자 수의 유일성, 운용보수의 허용 범위 및 만기일과 설정일의 선후관계 등을 검증 규칙으로 정의하였다. 이러한 판정은 LLM의 확률적 응답이 아니라 사전에 명시된 규칙에 따라 수행되므로 동일한 입력에 동일한 결과를 제공한다.

## 2.2 기존 연구와의 차이 및 본 연구의 차별성

기존 온톨로지 및 SHACL 기반 연구는 이미 구조화된 RDF 데이터가 스키마와 제약을 만족하는지 검증하는 데 주로 활용되어 왔다. 반면 일반적인 LLM 가드레일은 유해 응답, 개인정보, 출력 형식 또는 정책 위반 등 모델 입출력의 안전성을 통제하는 데 초점을 둔다(Dong et al., 2024). 즉 기존 접근에서 온톨로지는 구조화 데이터의 규칙 검증, 가드레일은 LLM 입출력 필터링이라는 서로 다른 문제를 다루는 경우가 많다.

본 연구는 두 접근을 단순히 병렬 배치하지 않고 **비정형 금융문서의 교차검증 과정 안에서 역할별로 연결**하였다. 첫째, LLM이 신탁계약서와 IM에서 추출한 `raw_text`를 ABox로 변환하여, SHACL 검증을 LLM 문서 추출 결과에 적용하였다. RDF에는 LLM 후보값인 `value`와 `unit`이 아니라 근거 원문인 `raw_text`만 저장한다. 둘째, SHACL은 필수 개념·관계와 업무 규칙의 위반을 진단하고, G1·G2·G3 가드는 오류값을 재시도·반려·제거하는 집행 기능을 담당하도록 분리하였다. 셋째, 온톨로지 검증만으로는 판단하기 어려운 두 문서의 의미적 동등성은 결정론적 파싱과 값 비교를 우선 적용한 뒤, 미해결 항목에만 LLM Judge를 선택적으로 사용하였다.

따라서 본 연구의 독창성은 온톨로지나 가드 자체를 새롭게 제안한 데 있지 않다. **신탁계약서 도메인에 특화된 4개념·14개 필드 구조, SHACL 진단과 가드 집행의 분리, 결정론적 비교와 선택적 LLM의 역할 배분, 그리고 각 구성의 효과를 조건별 반복실험과 통계로 평가한 통합 프레임**에 차별성이 있다. 이는 확률적 LLM의 추출·의미 해석 능력과 명시적 규칙의 재현성·설명 가능성을 결합한 neuro-symbolic 검증 구조에 해당한다.

# 3. 연구방법

## 3.1 하네스 파이프라인

본 연구의 하네스는 `PDF 입력 → LLM 구조화 추출 → 가드 검증 → RDF/ABox 변환 → SHACL 진단 → 결정론적 파싱·교차비교 → 선택적 LLM Judge → 채점·통계 분석`의 순서로 구성하였다.

먼저 멀티모달 LLM이 두 문서를 읽고 사전에 정의된 스키마에 따라 각 필드의 `value`, `unit`, `raw_text`, `citation`을 추출한다. `value`와 `unit`은 LLM이 제시한 후보값이며, 검증 근거로는 문서 원문인 `raw_text`와 문서명·페이지를 포함하는 `citation`을 사용한다. 두 문서의 결과는 같은 필드 아래 `contract`와 `im`으로 배치하여 비교 가능한 구조로 만든다.

추출 결과에는 세 종류의 결정론적 가드를 적용한다. **G1 형식 가드**는 응답이 유효한 JSON이고 Pydantic 스키마를 충족하는지 확인하며, 오류가 발생하면 재시도하거나 분석을 중단한다. **G2 출처 가드**는 인용 페이지가 실제 PDF 범위 안에 존재하는지 검사하여 가짜 인용을 탐지하고 해당 값을 무효화한다. **G3 제약 가드**는 보수율의 허용 범위와 만기일이 설정일보다 늦어야 한다는 관계 등 수치·논리 조건을 검사하고 위반값을 reject 또는 null 처리한다.

가드 처리 결과는 계약서와 IM별 `Fund`, `Party`, `FeeSchedule`, `RedemptionTerms` 인스턴스로 변환된다. 매핑 코드는 각 필드의 `raw_text`만 RDF 리터럴로 저장하며, 원문 근거가 없는 필드는 트리플을 생성하지 않는다. 생성된 ABox는 TBox와 SHACL 규칙으로 검증한다. SHACL은 데이터가 필수 구조와 업무 규칙을 충족하는지 **진단**하고, 가드는 확인된 오류를 후속 단계로 전달하지 않도록 **집행**한다. 일부 수치·날짜 규칙이 중복되더라도 SHACL은 위반 리포트를 생성하고, G3는 위반 필드를 실제로 null 처리한다.

이후 계약서와 IM의 원문을 코드로 교차비교한다. 원문이 같으면 일치, 한쪽 근거가 없으면 `missing_evidence`, 양쪽 원문이 다르면 `needs_review`로 분류한다. 다만 “연 1,000분의 8.9”와 “연 0.89%”처럼 표현만 다른 사례를 구분하기 위해 `parse_percent`, `parse_date`, `parse_duration`, `parse_boolean` 등의 필드별 결정론적 파서로 값을 표준 형태로 해석한 뒤 비교한다. 결정론적 비교 후에도 펀드 정식명과 약칭, 상품 유형 및 서술형 환매조건처럼 의미적 동등성을 확정할 수 없는 항목에만 LLM Judge를 적용한다. 따라서 LLM은 최초 추출에는 사용되지만, 비교 단계에서는 결정론적 판정을 우선하고 미해결 항목에만 선택적으로 사용된다.

## 3.2 3-Tier 실험 및 통계 방법

최종 실험에서는 세 티어 모두 G1·G2·G3 가드를 공통으로 적용하고, **비교 엔진과 LLM 사용 범위**만 달리하였다. Tier 1(`ontology_policy`)은 `parse_percent`, `parse_date`, `parse_duration`, `parse_boolean` 등의 결정론적 파서와 필드별 비교 정책만 사용한다. Tier 2(`ontology_policy_judge`)는 Tier 1의 결정론적 비교를 먼저 수행하고, 결론을 내리지 못해 `needs_review`로 남은 항목에만 필드 전용 LLM Judge를 선택적으로 호출한다. Tier 3(`harness_norm_judge`)는 모든 평가 사례에 LLM 정규화를 적용하고 남은 검토 항목을 LLM Judge로 판정한다. 세 조건의 모델과 입력은 동일하게 유지하여 LLM의 투입 범위에 따른 차이를 비교하였다.

하네스의 판정은 정답 라벨과 비교하여 TP·FP·FN·TN으로 집계하며, 양성은 계약서와 IM의 **불일치**로 정의하였다. 불일치를 놓치는 FN의 비용이 헛알람인 FP보다 크다는 업무 특성을 반영하여 재현율에 두 배의 가중치를 부여한 F2를 주 지표로 사용하고, 정밀도·재현율·F1 및 업무 효율성 점수(EI)를 함께 산출하였다.

각 티어는 30회 반복 실행하였다. Tier 2와 Tier 3의 차이는 분산이 서로 다를 수 있음을 고려한 **Welch 독립표본 t검정**과 **Mann-Whitney U 검정**으로 분석하고, 차이의 크기는 **Cohen's d**로 확인하였다. Tier 1은 LLM Judge를 사용하지 않아 반복 결과가 동일한 결정론적 기준선이므로, Tier 2·3과의 비교에는 Tier 1의 점수를 기준값으로 둔 **일표본 t검정**을 사용하였다. 티어별 평균, 표본표준편차 및 t 분포 기반 95% 신뢰구간을 산출하였으며, 구체적인 성능 수치와 조건별 우열은 분석 결과에서 제시한다.
